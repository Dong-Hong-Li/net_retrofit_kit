# 快速开始

**一句话：** 加依赖 → 在 `main()` 里设置 `NetRequest.options` → 定义带 `@NetApi()` 的抽象类 → 执行 build_runner → 使用 `XxxImpl()`。

---

## 1. 依赖

```yaml
dependencies:
  net_retrofit_kit: ^0.1.0
  dio: ">=5.0.0"
dev_dependencies:
  build_runner: ^2.4.0
```

执行 `flutter pub get`。

---

## 2. 配置（一次）

```dart
void main() {
  WidgetsFlutterBinding.ensureInitialized();
  NetRequest.options = const NetOptions(
    baseUrl: 'https://api.example.com',
    connectTimeout: Duration(seconds: 30),
    receiveTimeout: Duration(seconds: 30),
    sendTimeout: Duration(seconds: 30),
  );
  runApp(const MyApp());
}
```

必须在任意请求前执行。

---

## 3. 定义 API

- 抽象类 + `@NetApi()`。
- 方法：`@Get(path)`、`@Post(path)` 等。
- 参数按需：`@Body()`、`@QueryKey('name')`、`@Path('id')`、`@Header('name')`。
- 返回：`Future<T?>`（T 需有 `fromJson`）或 `Future<bool?>`。
- 静态 getter 返回生成类：`UserApiImpl`。

```dart
part 'user_api.g.dart';

@NetApi()
abstract class UserApi {
  static UserApi get instance => UserApiImpl();
  @Get('/user/info')
  Future<UserModel?> getUserInfo();
  @Post('/login')
  Future<AuthModel?> login(@Body() LoginRequest body);
  @Get('/user/:id')
  Future<UserModel?> getUser(@Path('id') String id);
}
```

---

## 4. 生成

```bash
dart run build_runner build --delete-conflicting-outputs
```

会生成 `user_api.g.dart` 和 `UserApiImpl`。改 API 或注解后需重新执行。

---

## 5. 使用

```dart
await UserApi.instance.getUserInfo();
await UserApi.instance.login(LoginRequest(phone: '138'));
await UserApi.instance.getUser('123');
```
