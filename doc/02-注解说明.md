# 注解说明

**速查表：**

| 注解 | 作用范围 | 用途 |
|------|----------|------|
| `@NetApi(client?, responseType?, unwrapSuccess?)` | 类 | 可选 client，不传时走 [NetRequest.defaultKey](04-多Client.md#defaultkey)；可选响应类型、成功只返回 data |
| `@Get(path)` `@Post(path)` `@Put(path)` `@Delete(path)` | 方法 | HTTP 方法 + 路径；可选 `contentType: ContentType.formData` |
| `@Body()` | 参数 | 请求体 |
| `@Query()` | 参数 | 完整 query map |
| `@QueryKey('name')` | 参数 | 单个 query 参数 |
| `@Path('id')` | 参数 | 路径段；path 中写 `:id` |
| `@Header('Authorization')` | 参数 | 请求头 |
| `@DataPath('key')` | 方法 | 从 `response.data['key']` 解析 |
| `@Part('file')` | 参数 | 多部分表单（配合 `ContentType.formData`） |
| `@StreamResponse()` | 方法 | 返回流（SSE / 按行） |
| `[CallOptions? options]`（可选位置参数） | 方法 | 单次请求选项：cancelToken、clientKey；放 `[]` 以与 `{}` 接口参数区分。若方法需其它命名参数则用 `{CallOptions? options}`（Dart 不允许同方法同时用 `[]` 与 `{}`） |

---

## @NetApi（类）

```dart
@NetApi()  // 或 @NetApi(client: 'upload', responseType: 'BaseResponse', unwrapSuccess: true)
abstract class UserApi { ... }
```

| 参数 | 含义 |
|------|------|
| `client` | 具名 Client key；需用 `NetRequest.setClient('upload', client)` 注册。 |
| `responseType` | 解析用响应类型名。 |
| `unwrapSuccess` | `true` 只返回 `response.data`；`false` 返回完整 response。 |

---

## HTTP 方法 + 路径

```dart
@Get('/user/list')
@Post('/user', contentType: ContentType.formData)  // 上传
@Put('/user/:id')
@Delete('/user/:id')
```

路径相对 `NetRequest.options.baseUrl`。

---

## Body、Query、Path、Header

```dart
@Post('/login')
Future<Auth?> login(@Body() LoginRequest body);

@Get('/search')
Future<List?> search(@QueryKey('keyword') String q, @QueryKey('page') int page);

@Get('/user/:id')
Future<User?> getUser(@Path('id') String id);

@Get('/me')
Future<User?> me(@Header('Authorization') String token);
```

---

## DataPath（从嵌套 key 解析）

```dart
@Get('/nested')
@DataPath('result')
Future<Nested?> getNested();
```

---

## Part（多部分 / 文件上传）

```dart
@Post('/upload', contentType: ContentType.formData)
Future<Result?> upload(@Part('file') File file, @Part('name') String name);
```

---

## StreamResponse（流 / SSE）

```dart
@Get('/stream/3')
@StreamResponse()
Future<Stream<String>> getStreamLines({CallOptions? options});
```

## CallOptions? options（请求级选项，区分其它具名参数）

用**唯一具名参数** `CallOptions? options` 传「请求级选项」，与 body、query、回调等其它具名参数区分。

### 生成器逻辑

- **识别**：方法上存在具名参数且类型为 `CallOptions?`、参数名为 **`options`** 时生效。
- **普通请求**（`requestHttp`）：生成 `clientKey: options?.clientKey`（若类上有 `@NetApi(client: 'xxx')` 则生成 `options?.clientKey ?? 'xxx'`）、`cancelToken: options?.cancelToken`。
- **流式请求**（`requestStreamResponse`）：同上，生成 `clientKey`、`cancelToken` 透传。
- 未传 `options` 时上述参数为 null，行为与不传一致。

### 接口与案例

**接口定义（每个方法都可加 `{CallOptions? options}`）：**

```dart
@NetApi()
abstract class DemoServer {
  @Get('/get')
  Future<DemoModel?> getUserInfo({CallOptions? options});

  @Post('/post')
  Future<DemoModel?> login(@Body() Map<String, dynamic> body, {CallOptions? options});

  @Get('/stream/3')
  @StreamResponse()
  Future<Stream<String>> getStreamLines({CallOptions? options});
}
```

**调用示例（可选传、取消、client）：**

```dart
// 不传 options
await api.getUserInfo();

// 传 cancelToken 便于取消
final token = CancelToken();
await api.getStreamLines(options: CallOptions(cancelToken: token));
token.cancel('用户取消');

// 指定 clientKey
await api.getUserInfo(options: CallOptions(clientKey: 'upload'));
```

**页面中流式请求 + 取消（StreamRequestPage 案例）：**

```dart
CancelToken? _cancelToken;
final _repository = DemoRepository.instance;

Future<void> _startStream() async {
  _cancelToken = CancelToken();
  final result = await _repository.fetchStreamLines(
    options: CallOptions(cancelToken: _cancelToken),
  );
  // ...
}

void _cancelStream() {
  _cancelToken?.cancel('Cancelled by user');
}

@override
void dispose() {
  _cancelToken?.cancel();
  super.dispose();
}
```
