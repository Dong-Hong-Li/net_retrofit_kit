# 快速开始

从零把 net_retrofit_kit 接入 Flutter 项目的完整步骤。

---

## 1. 添加依赖

在项目的 [pubspec.yaml](https://github.com/Dong-Hong-Li/net_retrofit_kit/blob/main/pubspec.yaml) 中：

```yaml
dependencies:
  net_retrofit_kit: ^0.1.0
  dio: ">=5.0.0"

dev_dependencies:
  build_runner: ^2.4.0
```

本地开发可用：`net_retrofit_kit: path: ../net_retrofit_kit`。

执行：

```bash
flutter pub get
```

---

## 2. 在发起请求前配置

在任意请求之前（通常在 `main()`）设置 [NetRequest.options](https://github.com/Dong-Hong-Li/net_retrofit_kit/blob/main/lib/src/network/net_request.dart)：

```dart
import 'package:net_retrofit_kit/net_retrofit_kit.dart';

void main() {
  WidgetsFlutterBinding.ensureInitialized();
  NetRequest.options = const NetOptions(
    baseUrl: 'https://api.example.com',
    connectTimeout: Duration(seconds: 30),
    receiveTimeout: Duration(seconds: 30),
    sendTimeout: Duration(seconds: 30),
  );
  runApp(const MyApp());
}
```

未设置就发起请求会抛出 `StateError`。更多配置（拦截器等）见 [docs/03-配置.md](https://github.com/Dong-Hong-Li/net_retrofit_kit/blob/main/docs/03-配置.md)。

---

## 3. 定义 API（抽象类 + 注解）

新建一个 Dart 文件（如 `lib/api/user_api.dart`），用 [注解](https://github.com/Dong-Hong-Li/net_retrofit_kit/blob/main/lib/src/generate/annotations.dart) 定义**抽象**类：

```dart
import 'package:net_retrofit_kit/net_retrofit_kit.dart';

part 'user_api.g.dart';

@NetApi()
abstract class UserApi {
  static UserApi get instance => UserApiImpl();

  @Get('/user/info')
  Future<UserModel?> getUserInfo();

  @Post('/login')
  Future<AuthModel?> login(@Body() LoginRequest body);

  @Get('/user/:id')
  Future<UserModel?> getUser(@Path('id') String id);

  @Get('/search')
  Future<List<Item>?> search(@QueryKey('keyword') String keyword, @QueryKey('page') int page);
}
```

- **类**：必须是 `abstract`，并加 `@NetApi()`；若用具名 Client 可写 `@NetApi(client: 'upload')`。
- **方法**：用 `@Get(path)`、`@Post(path)` 等标注；参数按需用 `@Body()`、`@Query()`、`@QueryKey(name)`、`@Path(name)`、`@Header(name)`。
- **返回值**：`Future<T?>`，其中 `T` 需有 `fromJson` 用于解析；或 `Future<bool?>` 仅表示成功与否。

静态 getter（如 `instance`）应返回**生成类**名（`UserApiImpl`）。生成器会生成名为 `{类名}Impl` 的类，放在同文件的 `.g.dart` 中。

---

## 4. 生成实现

在**应用**根目录（有 `pubspec.yaml`、`build.yaml` 的目录）执行：

```bash
dart run build_runner build --delete-conflicting-outputs
```

会生成 `user_api.g.dart`（或根据 [build.yaml](https://github.com/Dong-Hong-Li/net_retrofit_kit/blob/main/build.yaml) 生成 `.net_retrofit_kit.g.part`），内含 `UserApiImpl`，内部调用 `NetRequest.requestHttp`（或 `@NetApi(client: '...')` 指定的 Client）。

修改注解或方法签名后需再次执行该命令。

---

## 5. 在业务中使用

```dart
final user = await UserApi.instance.getUserInfo();
final auth = await UserApi.instance.login(LoginRequest(phone: '13800138000'));
final u = await UserApi.instance.getUser('123');
final list = await UserApi.instance.search('flutter', 1);
```

无需手写实现类或重复的 `NetRequest.requestHttp` + `BaseResponse.fromJson` 样板代码。
